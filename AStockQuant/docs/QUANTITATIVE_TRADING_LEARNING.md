## 一、核心交易原则 (海龟交易法则)

### 1.1 四大核心原则
```
1. 掌握优势 — 找到正期望值策略，长期坚持
2. 管理风险 — 单笔交易≤2%账户风险
3. 坚定不移 — 完全机械执行，杜绝情绪干扰
4. 简单明了 — 简单系统更易执行
```

### 1.2 入市系统 (Donchian Channel突破)
```
系统一 (短期): 20日突破
  - 价格 > 20日最高价 → 做多
  - 价格 < 20日最低价 → 做空

系统二 (长期): 55日突破
  - 更保守，信号更少但更可靠
```

### 1.3 离市规则
```
系统一退出: 10日反向突破
系统二退出: 20日反向突破
```

### 1.4 止损规则
```
止损位 = 入场价 ± 2×ATR
ATR = Average True Range (20日均值)
```

### 1.5 头寸管理 (金字塔加仓)
```
头寸单位 = 账户风险 / (ATR × 合约乘数)
加仓间隔: 每0.5ATR波动加仓一次
最多加仓4次
```

## 二、动量策略核心

### 2.1 动量公式
```
动量 = (当前价 - N期前价) / N期前价
N通常 = 20日或60日
```

### 2.2 动量交易规则
```
动量 > 0 → 买入/持有
动量 < 0 → 卖出/空仓
```

### 2.3 RPS (相对价格强度)
```
RPS = 个股收益率在群体中的排名百分位
只选RPS > 70的标的
```

## 四、量化交易常见错误

### 4.1 我之前的错误
1. **过度复杂化** — 引入太多指标导致信号冲突
2. **忽视止损** — 没有严格执行2%风险限制
3. **预测而非跟随** — 试图预测市场而非跟随趋势
4. **忽视仓位管理** — 没有根据ATR计算头寸

### 4.2 正确做法
```python
# 正确流程
1. 确认趋势 (价格 > 20日均线)
2. 突破入场 (价格 > 20日最高价)
3. ATR计算仓位 (单笔风险≤2%)
4. 金字塔加仓 (每0.5ATR)
5. 移动止损 (让利润奔跑)
6. 反向突破离场 (跌破10日最低价)
```

## 五、ETF量化优势

1. **数据纯净** — 一篮子股票，抗噪音
2. **交易成本低** — 免印花税，T+0
3. **流动性好** — 适合程序化
4. **趋势明显** — 宽基ETF更能反映宏观趋势
